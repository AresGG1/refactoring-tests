### Рекомендації для рефакторингу

1. В методі `statement()` використовуються числа (2, 1.5, 3) без пояснення їх значення.

2. Використання оператора `+=` для рядків у циклі створює нові об'єкти на кожній ітерації.

3. `Movie` є публічним, тоді як `Rental` та `Customer` пакетні.

4. Використано анотацію `@SuppressWarnings`, приховує проблему.

5. Метод `statement()` одночасно обчислює суми та формує текстовий звіт, краще розділити.

6. Метод `getTitle ()` має зайвий пробіл перед дужками.

7. `MovieType` всередині класу `Movie`, краще винести в окремий файл.

8. Список `rentals` у конструкторі `Customer` може бути змінений ззовні після створення об'єкта.
